Куда писать стили

link, href, rel

Устанавливает связь с внешним документом вроде файла со стилями или со шрифтами. Элемент link обычно размещается внутри контейнера head

Тег одиночный и имеет два атрибута: href и rel. Первый указывает путь к файлу, а второй определяет отношения между документом и файлом. Значение для ссылок на css-файл: stylesheet.

Для html-страницы можно использовать несколько таблиц стилей. Таблицу соответственно можно использовать в нескольких html-файлах, чем она и отличается от тега style

style

Этот тег пишется внутри head. Внутри него можно писать стили, которые будут пременены только для текущего документа.

Селекторы

Запись стиля состоит из двух частей: элемента веб-страницы — селектора, и команды форматирования — блока объявления.

Селектор сообщает браузеры какой элемент форматировать, а в блоке объявления перечисляются форматирующие команды из двух частей: свойство и значение.

Для одного селектора можно задать несколько свойств через точку с запятой.

Универсальный селектор

Применяется сразу ко всем элементам.

Селектор тега

В качестве селектора указывается название тега без скобок.

Селектор класса

В качестве селектора указывается название класса с точкой перед. Это позволит применить стиль ко всем элементам с таким классом. Также селектор можно задать для конкретных элементов с таким классом.

Селектор ID

Также как с классами, но через #.

Дочерний cелектор

Свойство применяется к определённым элементам внутри определённого элемента.

Селектор соседа

Свойство применяется к определённому элементу, стоящему после определённого элемента.

Селектор соседей

Также, но через ~. Распростроняется на все элементы указанного вида после.

Каскадность

Стили для элемента могут быть определены в нескольких местах: внутри одного файла стилей и в разных файлах стилей.

Браузер находит все CSS-правила, затрагивающие данный элемент, а затем комбинирует их и получает итоговый список свойств для этого элемента. Комбинирование свойств производится по четким правилам, которые опираются на приоритетность и специфичность.

Название “Каскадность” появилось из-за описанного механизма комбинирования стилей из разных css-правил.
