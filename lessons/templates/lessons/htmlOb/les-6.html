Разметка страницы

Разделы страницы

header

Тег header предназначен для создания «шапки» страницы сайта в целом или для «шапки» раздела страницы. В header, eсли он выступает в роли шапки страницы, помещают логотип или название сайта, слоган, тематическое изображение, контакты, форму поиска, иконки соц. сетей и т.п. Если же header используется как шапка раздела страницы, в него помешают заголовок раздела.

nav

Тег nav представляет раздел страницы, содержащий ссылки на другие страницы или на разделы текущей страницы.

В тех случаях, когда содержимое элемента nav представляет собой список элементов, используйте списки для разметки навигации.

Не все группы ссылок на странице должны быть в элементе nav этот элемент прежде всего предназначен для разделов, состоящих из больших блоков навигации, т.е. для главной навигации по сайту между его основными разделами, но не для второстепенной (не для перемещения между подразделами и страницами большого раздела).

main

Тег main представляет основное (главное) содержимое страницы сайта.

Область основного содержимого документа включает в себя содержимое, которое уникально для этого документа и исключает содержимое, повторяющееся в наборе нескольких документов, такое как шапка сайта, навигация, подвал, боковая колонка и т.п.

В документе не может быть более одного элемента main.

Нельзя помещать элемент main в элементы article, aside, footer, header или nav.

footer

Тег footer используют для создания подвала страницы сайта в целом или раздела страницы. Футер обычно содержит следующую информацию о своём разделе: кто написал его, ссылки на документы, авторские права и т. п.

aside

Тег aside используют для оформления второстепенного содержимого страницы: навигация 2 уровня (локальная навигация или навигация по разделу), ссылки по теме, фильтр товаров и т.п.

Чаще всего данный элемент используется для создания боковой колонки или врезки.

Не следует использовать элемент aside просто для выделения части информации в основном содержимом.

article

Тег article представляет полную, или самостоятельную, композицию в документе. Это может быть статья на форуме, в журнале или газете, запись в блоге, отправленный пользователем комментарий, интерактивный виджет или любой другой независимый объект содержимого.

Когда элементы article вложены друг в друга, внутренние article представляют статьи, которые в принципе относятся к содержимому внешнего article. Например, статья в блоге на сайте и комментарии к ней: можно представить эти комментарии как элементы article, вложенные в элемент article этой статьи блога.

Когда главное содержимое страницы (т. е. исключая подвал, шапку, блок навигации и боковую панель) целиком представляет собой самостоятельную композицию, содержимое следует размечать с использованием тега main.

section

Тег section представляет раздел документа. Раздел, в данном случае, — это тематическое группирование содержимого. Тема каждого элемента section должна быть указана путём включения заголовков (h1-h6) в section.

Вспомогательные элементы разметки

div

Тег div не несет логической нагрузки. Этот тег используется для группировки других элементов html-страницы.

Рекомендуется использовать div, когда ни один другой элемент не подходит.

Элементы div могут пригодиться для стилистических целей.
